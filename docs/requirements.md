# maki 要件定義

## 概要

**maki** は、日常の定型業務を自動監視・実行する軽量パッシブAI。ユーザーに代わって各種タスクを実行し、必要に応じて人間に確認を取る。

## 機能要件

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| F1 | **常駐監視** | Must | バックグラウンドで定期的に各監視対象を巡回する |
| F2 | **Slack監視・応答** | Must | Slackのメンションを検出し、応答案を提示。承認後に返信する |
| F3 | **実行前の確認** | Must | 自動実行する前にユーザーに確認を取る仕組み |
| F4 | **ループ制御** | Must | 次にアクティブになるタイミングを2種類持つ。デフォルトのタイマー間隔と、ユーザーが指定した時刻・間隔 |
| F5 | **アイドル時の問いかけ** | Must | 監視対象にタスクが無い場合、ユーザーに「何かやることありますか？」と聞く |
| F6 | **Issue取得・作業準備** | Should | アサインされたIssueの内容を取得し、作業に必要なコンテキストを整える |
| F7 | **Issue実装指示** | Should | 可能であればIssueの実装まで行う |
| F8 | **勤怠自動入力** | Should | 勤怠システムへの出退勤入力を自動化する |
| F9 | **手動タスク実行** | Should | ユーザーがその場で任意のタスクを指示できる |
| F10 | **コンテキスト・スキル管理** | Should | AIに渡すコンテキストの設計やスキル定義を自分で管理・調整できる |
| F11 | **音声入力** | Stretch | 音声でタスク指示を受け付ける |

## 制約・方針

- タスクの実行手段は **スキル（プラグイン）** に依存する。どのスキルを使うかは設定で管理する
- 本体は軽量に保つ。シンプルなループで動作する
- 人間の確認は最も簡単な方法（ターミナル上）で行う
